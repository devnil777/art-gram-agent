# HTML Report Generation Guide

## Обзор

Система отчётов переведена на HTML формат с группировкой событий по датам. Каждый день получает отдельный красивый HTML файл, плюс сводная страница со всеми датами.

## Структура отчётов

### Файлы
```
output/
  events_report_summary.html     ← сводка со всеми датами
  2026-04-20.html                ← события на 20 апреля 2026
  2026-04-21.html                ← события на 21 апреля 2026
  2026-04-25.html                ← события на 25 апреля 2026
  events_report_unknown.html     ← события без даты (если есть)
```

### Сводная страница (summary.html)

**Содержит:**
- Общие статистика (всего событий, уникальных дат, каналов)
- Список всех дат с количеством событий
- Ссылки на дневные отчёты
- Быстрая навигация по датам

**Назначение:** Обзор и быстрый поиск нужной даты

### Дневной отчёт (YYYY-MM-DD.html)

**Содержит для каждого дня:**
- Отформатированная дата (例: "20 апреля 2026")
- Количество событий в этот день
- Статистика (основной тип события, средняя уверенность)
- Полный список событий с детальной информацией

**Для каждого события показывается:**
- ✨ **Название** (с указанием типа)
- 📝 **Краткое описание** (если доступно)
- 📍 **Место проведения**
- 🕐 **Время события**
- ✓ **Уверенность** (визуальные звёздочки: ★★★★★)
- 🔗 **Ссылка на канал** (с названием канала)
- 🔗 **Ссылка на сообщение в Telegram**

## Примеры использования

### Генерация отчётов (по умолчанию)

```bash
python process_runner.py
```

**Результат:**
- Создаёт/обновляет HTML файлы в `output/`
- Один файл на день для каждого дня, содержащего события
- Автоматически обновляет сводную страницу

### Без генерации отчётов

```bash
python process_runner.py --skip-report
```

### Полная переобработка с отчётами

```bash
python process_runner.py --full
```

### 📱 Мобильная оптимизация

- Адаптивная сетка для разных размеров экранов

- Компактная компоновка на мобильных устройствах

- **Звёздочки уверенности** ★★★★☆ для быстрого визуального восприятия

## Группировка событий

### По датам

События группируются по датам из полей:
1. `start_datetime` (приоритет) — рассчитанная дата начала события
2. `message_date` — дата сообщения в чате (fallback)

Если оба поля пусты → событие попадает в раздел "События без даты"

### Внутри дня

События сортируются по:
1. **Уверенности** (по убыванию: 5 звёзд → 1 звезда)
2. По типу события

## Форматирование дат

### Отображение

- **Входные данные**: `2026-04-20`
- **Отображение**: `20 апреля 2026`

Автоматическое преобразование в человечески читаемый формат с русскими названиями месяцев.

### Типы событий

- `exhibition` → 🎨 Выставка
- `tour` → 🚶 Тур / Экскурсия
- `talk` → 🎤 Лекция / Доклад
- `workshop` → 🛠️ Мастер-класс
- `meetup` → 👥 Встреча
- `presentation` → 📺 Презентация
- `other` → ❓ Другое

## Статистика на странице

### Сводная страница показывает:
- **Всего событий** — сумма по всем датам
- **Уникальных дат** — количество дней с событиями
- **Каналов** — количество источников
- **Ошибок** — количество сообщений, которые не удалось обработать

### Дневная страница показывает:
- **Событий** — в этот день
- **Основной тип** — самый частый тип события в этот день
- **Средняя уверенность** — среднее значение уверенности

## Расширенные возможности

### 📊 Аналитика

Все данные сохраняются в `processed/` в JSONL формате, поэтому можно:
- Анализировать события программно
- Строить собственные отчёты
- Интегрировать с внешними системами

### 🔄 Переиспользование данных

HTML отчёты генерируются из persistent storage, поэтому:
- Можно перегенерировать отчёты без переобработки сообщений
- Можно строить отчёты за конкретный период
- Можно менять стили без пересчёта данных

### 🌍 Интеграция

HTML файлы можно:
- Открыть в браузере как файлы (file://)
- Загрузить на веб-сервер
- Встроить в веб-приложение
- Отправить по email (как вложение с кодировкой base64)

## Технические детали

### Кодировка

- **UTF-8** для поддержки русского языка
- **HTML5** doctype для современной совместимости
- **CSS Grid и Flexbox** для адаптивности

### Производительность

- **Статические HTML файлы** — быстрое открытие в браузере
- **Нет JavaScript** — не требует выполнения кода
- **Минимальный размер** — обычно 8-10 KB на день

### Локализация

Все текстовые элементы на **русском языке**:
- Названия месяцев
- Типы событий
- Описания полей
- Сообщения об ошибках

## Часто задаваемые вопросы

**Q: Как открыть отчёты?**
A: Просто откройте HTML файл в браузере (Chrome, Firefox, Safari и т.д.)

**Q: Можно ли менять стили?**
A: Да, стили встроены в HTML. Измените `<style>` блок в `report_generator.py`

**Q: Почему у события нет даты?**
A: Если LLM не смог определить дату события, оно попадёт в раздел "События без даты"

**Q: Как долго генерируются отчёты?**
A: Обычно менее 1 секунды для 100 событий

**Q: Сколько места занимают отчёты?**
A: ~10 KB на день (в зависимости от количества событий)
